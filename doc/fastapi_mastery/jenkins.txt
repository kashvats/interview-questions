================================================================================ -- MODULE 01: JENKINS ARCHITECTURE - CONTROLLER VS AGENTS
Topic: The Master-Slave shift, System internals, and Distributed builds
Project: Zomato-Infra Jenkins Setup -- 1. CONCEPT EXPLANATION (Hinglish) -- Dosto, Jenkins ek "Distributed" system hai. Isme sara kaam ek hi computer par nahi hota.

- **Jenkins Controller (Master)**: Ye system ka brain hai. Ye configuration manage karta hai, jobs schedule karta hai, aur UI dikhata hai. (Lekin ye actual build nahi karta!).
- **Jenkins Agents (Node/Slave)**: Ye wo machines hain jahan actual code compile, test aur deploy hota hai. 
- **Staff Level Tip**: Kabhi bhi controller par build execute nahi karni chahiye (Zero-executor on master). Isse controller hang nahi hota aur stability bani rehti hai. -- MODULE 02: INSTALLATION & AGENT CONNECTIVITY

- **Connecting Agents**:
 - **SSH**: Controller agent ke andhar login karta hai. (Standard for Linux nodes).
 - **JNLP/Inbound**: Agent controller ko request bhejta hai. (Useful for nodes behind a firewall). -- MODULE 03: JOB TYPES - FREESTYLE VS PIPELINE

2. CODE: A PRODUCTION-GRADE DECLARATIVE PIPELINE -- ```groovy
pipeline {
 agent any

 parameters {
 choice(name: 'DEPLOY_ENV', choices: ['dev', 'staging', 'prod'], description: 'Choose Env')
 string(name: 'APP_VERSION', defaultValue: '1.0.0', description: 'Enter Version')
 }

 environment {
 APP_NAME = "zomato-delivery"
 DB_URL = credentials('prod-db-url')
 }

 stages {
 stage('Checkout') {
 steps {
 git branch: 'main', url: 'https://github.com/zomato/delivery-api.git'
 }
 }
 stage('Build') {
 steps {
 echo "Building version ${params.APP_VERSION} for ${params.DEPLOY_ENV}"
 sh './gradlew clean build'
 }
 }
 }

 post {
 failure {
 echo "Deployment Failed. Notifying Dev team..."
 }
 always {
 cleanWs()
 }
 }
}
```

7. STAFF-LEVEL PRO TIPS (PART 1) -- - **cleanWs()**: Hamesha `post` block mein use karein taaki workspace full na ho jaye aur naye builds clean start hon.
- **Label Agents**: Hamesha nodes ko labels dein (e.g., `linux-high-cpu`, `mac-ios-build`). `any` use karna production mein bad practice hai.
- **Build Discarder**: `properties` block use karein taaki purane builds delete ho jayein aur disk save ho.
