MODULE 02: CORE ABSTRACTIONS & SCHEMA
Topic: BaseMessage, Document, and Runnable Interface (LCEL)
Project: Zomato-AI Communication Layer
--------------------------------------------------------------------------------

1. CONCEPT EXPLANATION (Hinglish)
--------------------------------------------------------------------------------
Dosto, LangChain mein sab kuch "Messages" aur "Runnables" hai.

- **BaseMessage types**: 
 - `HumanMessage`: User ka input.
 - `AIMessage`: Model ka response (including tool calls).
 - `SystemMessage`: Model ko instruction dena (e.g. "You are a Zomato Agent").
 - `ToolMessage`: Tool execution ka result model ko wapas bhejba.
- **Runnable Interface**: Sabse important concept. Har major class (LLM, Prompt, Parser) `Runnable` ko inherit karti hai. Iska matlab hai ki unhe `invoke()`, `batch()`, aur `stream()` ke through call kiya ja sakta hai.

**LCEL (LangChain Expression Language)**:
Ye "Pipe" operator (`|`) use karke complex flows banata hai. `Prompt | Model | Parser` ek standard flow hai.

2. CODE: THE CORE SCHEMA
--------------------------------------------------------------------------------
```python
from langchain_core.messages import HumanMessage, SystemMessage, AIMessage
from langchain_core.runnables import RunnableLambda

# 1. Message Schema
messages = [
 SystemMessage(content="You are a premium Zomato-AI concierge."),
 HumanMessage(content="What's the best Biryani in Indiranagar?")
]

# 2. Runnable Pattern
def upper_case(text: str) -> str:
 return text.upper()

runnable = RunnableLambda(upper_case)
# invoke, batch, and stream work automatically
result = runnable.invoke("biryani") 
print(result) # Output: BIRYANI
```

MODULE 05: EMBEDDINGS - THE VECTOR BRIDGE
Topic: Vector Dimensions, Performance vs Cost, and Caching
Project: Zomato-AI Dish Embeddings
--------------------------------------------------------------------------------

1. CONCEPT EXPLANATION (Hinglish)
--------------------------------------------------------------------------------
Dosto, "Every piece of text is a point in space." Embeddings text ko numbers (vectors) mein convert karti hain taaki hum semantic search kar sakein.

- **High Dimensions (e.g. 1536)**: Zyada details, par search slow aur costly hoti hai.
- **Performance Trade-off**: OpenAI's `text-embedding-3-small` vs `large`. Small version fast hai aur 90% cases mein Zomato search ke liye kafi hai.

**Caching Embeddings**:
Agar ek hi dish (e.g. "Chicken Biryani") ka description baar-baar embed ho raha hai, toh hum `CacheBackedEmbeddings` use karte hain taaki API costs 50% kam ho sakein.

2. CODE: EMBEDDING PIPELINE & CACHING
--------------------------------------------------------------------------------
```python
from langchain_openai import OpenAIEmbeddings
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore

# 1. Initialize Base Embeddings
underlying_embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 2. Setup Cache
store = LocalFileStore("./embedding_cache/")
cached_embedder = CacheBackedEmbeddings.from_bytes_store(
 underlying_embeddings, store, namespace=underlying_embeddings.model
)

# 3. Execution
vectors = cached_embedder.embed_documents(["Best Pizza in Delhi", "Butter Chicken lovers"])
```

7. STAFF-LEVEL PRO TIPS (PART 1)
--------------------------------------------------------------------------------
- **Version Pinning**: Anthropic ya OpenAI ke packages ko hamesha pin karein (`==`), warna breaking updates `Runnable` schema ko crash kar sakti hain.
- **Provider Agnostic**: Hamesha `ChatPromptTemplate` use karein taaki Claude se GPT par switch karna manual text formatting ke bina ho sake.
- **Cost Awareness**: Startup mein full model calls ki jagah `astream` aur `abatch` use karein parallel throughput ke liye.

MODULE 10: TEXT SPLITTERS & SEMANTIC CHUNKING
Topic: RecursiveCharacterTextSplitter, Token Limits, and Overlap Tuning
Project: Zomato-AI Policy Chunking
--------------------------------------------------------------------------------

1. CONCEPT EXPLANATION (Hinglish)
--------------------------------------------------------------------------------
Dosto, pure PDF ko ek sath prompt mein nahi daal sakte (Context Window Limit). Humein use "Chunks" mein todna padta hai.

- **RecursiveCharacterTextSplitter**: Sabse smart splitter. Pehle paragraphs par split karta hai, fir sentences par, fir words par, taaki semantic meaning lose na ho.
- **Chunk Size vs Overlap**:
 - `Chunk Size`: Ek chunk kitna bada hoga.
 - `Chunk Overlap`: Do chunks ke beech kitna context common hoga (essential for meaning continuity).
- **Token-based Splitters**: Kuch models (like OpenAI) fixed tokens support karte hain character length ki jagah. `tiktoken` splitter use karna yahan best practice hai.

2. CODE: PRODUCTION SPLITTING STRATEGY
--------------------------------------------------------------------------------
```python

splitter = RecursiveCharacterTextSplitter(
 chunk_size=1000,
 chunk_overlap=100,
 separators=["`n`n", "\n", ".", "!", "?", " ", ""]
)

docs = splitter.split_documents(raw_loaded_docs)
```

```python
vector_retriever = vectorstore.as_retriever()

# Ensemble: 50% BM25 + 50% Vector
ensemble_retriever = EnsembleRetriever(
 retrievers=[bm25_retriever, vector_retriever],
 weights=[0.5, 0.5]
)
```

MODULE 21: LANGGRAPH FUNDAMENTALS - NODES, EDGES & STATE
Topic: StateGraph, Nodes as Functions, and Conditional Edges
Project: Zomato-AI Multi-Step Order Workflow
--------------------------------------------------------------------------------

1. CONCEPT EXPLANATION (Hinglish)
--------------------------------------------------------------------------------
Dosto, "Chains are too rigid for the real world." 2026 mein serious AI applications LangGraph use karti hain kyunki ye **State Machine** principles par chalti hai.

- **State**: Ek shared "Notebook" (pydantic model) jo har node ke paas pass hoti hai.
- **Nodes**: Independent Python functions jo state ko modify karte hain (e.g. `check_inventory`, `calculate_bill`).
- **Edges**: Nodes ke beech ka rasta. Ye deterministic ho sakte hain ya `Conditional` (LLM decide karega ki next node kaunsa hoga).

2. CODE: THE BASIC STATE GRAPH
--------------------------------------------------------------------------------
```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated

# 1. Define State
class AgentState(TypedDict):
 messages: Annotated[list, add_messages]
 order_status: str

# 2. Define Nodes
def call_model(state: AgentState):
 # Logic to call LLM
 return {"messages": [response]}

# 3. Build Graph
workflow = StateGraph(AgentState)
workflow.add_node("agent", call_model)
workflow.set_entry_point("agent")
workflow.add_edge("agent", END)
app = workflow.compile()
```
