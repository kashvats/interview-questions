     1.8 HorizontalPodAutoscaler

       apiVersion: autoscaling/v2
       kind: HorizontalPodAutoscaler
       metadata:
       name: app-hpa
       namespace: production-app
       spec:
       scaleTargetRef:
       apiVersion: apps/v1
       kind: Deployment
       name: app
       minReplicas: 3 # Never scale below 3 replicas.
       maxReplicas: 10 # Never scale above 10 replicas.
       behavior:
       scaleUp:
       stabilizationWindowSeconds: 0 # No delay before scaling up.
       policies:
       - type: Percent
       value: 100 # Can grow by up to 100% per period.
       periodSeconds: 60
       scaleDown:
       stabilizationWindowSeconds: 300 # Wait 5 min before reducing replicas.
       policies:
       - type: Percent
       value: 50 # Reduce by at most 50% per period.
       periodSeconds: 60
       metrics:
       - type: Resource
       resource:
       name: cpu
       target:
       type: Utilization

       - type: Resource
       resource:
       name: memory

     1.9 NetworkPolicy

       apiVersion: networking.k8s.io/v1
       kind: NetworkPolicy
       metadata:
       name: app-network-policy
       namespace: production-app
       spec:
       podSelector:
       matchLabels:
       app: app
       policyTypes:
       - Ingress
       - Egress
       ingress:
       - from:
       - namespaceSelector: {} # Allows traffic from any namespace (tighten later).
       ports:
       - protocol: TCP
       port: 8080
       egress:
       - {} # Allow all outgoing traffic.



       ■ For stricter production, restrict ingress to only your ingress-nginx namespace, and egress to only DNS (53),
       your database, and Redis ports.

     3.1 Dockerfile (Multi-Stage)

       # syntax=docker/dockerfile:1.7
       FROM python:3.12-slim AS base
       ENV PYTHONDONTWRITEBYTECODE=1 # Prevent Python from creating .pyc files.
       ENV PYTHONUNBUFFERED=1 # Flush stdout/stderr immediately.
       ENV PIP_NO_CACHE_DIR=1 # Prevent pip from storing cache files.
       WORKDIR /app
       RUN apt-get update && apt-get install -y --no-install-recommends \
       build-essential \
       libpq-dev \
       netcat-openbsd \
       curl \
       && rm -rf /var/lib/apt/lists/*
       FROM base AS builder # Builder stage for installing Python dependencies.
       COPY requirements.txt .
       RUN pip install --upgrade pip && pip install -r requirements.txt
       FROM python:3.12-slim AS runtime # Final runtime stage — starts clean.
       ENV PYTHONDONTWRITEBYTECODE=1
       ENV PYTHONUNBUFFERED=1
       WORKDIR /app
       RUN apt-get update && apt-get install -y --no-install-recommends \
       libpq5 \
       curl \
       && rm -rf /var/lib/apt/lists/*
       RUN addgroup --system appgroup && adduser --system --ingroup appgroup appuser
       COPY --from=builder /usr/local /usr/local # Copy installed packages from builder.
       COPY . .
       COPY entrypoint.sh /entrypoint.sh
       COPY gunicorn.conf.py /app/gunicorn.conf.py
       RUN chmod +x /entrypoint.sh && \
       mkdir -p /app/staticfiles /app/media && \
       chown -R appuser:appgroup /app /entrypoint.sh
       USER appuser # Run the application as a non-root user.
       EXPOSE 8000
       HEALTHCHECK --interval=30s --timeout=5s --start-period=30s --retries=3 \
       CMD curl -f http://127.0.0.1:8000/health/ || exit 1
       ENTRYPOINT ["/entrypoint.sh"]
       CMD ["gunicorn", "-c", "/app/gunicorn.conf.py", "config.wsgi:application"]

     CHAPTER 06



     Final Blueprint & Operational Rules

     6.1 Final Architecture Blueprint

        React → Nginx container → Kubernetes Deployment
        Django → Gunicorn → Kubernetes Deployment
        Celery → Worker process → Kubernetes Deployment
        Celery Beat → Scheduler → Kubernetes Deployment (replicas: 1)
        Redis → Managed / dedicated → External service
        Postgres → Managed (RDS/Cloud SQL) → External service
        Media/Static → S3 + CDN → Object storage
        Terraform → Provisions all infra → IaC
        GitHub Actions → CI/CD pipeline → Test → Build → Push → Deploy
        Ingress → Public entrypoint → NGINX Ingress Controller + TLS
        Observability → Sentry + Prometheus + Grafana + centralised logs




     6.2 Operational Rules

     Concern                 Rule

     Migrations              Run as a Kubernetes Job — never hidden inside pod startup

     Celery Beat             Keep replicas: 1 unless you implement explicit leader election

     Workers                 Scale workers independently from web/API pods

     Backend HPA             Apply HPA to API pods; workers can have a separate HPA

     Postgres                Use managed RDS or Cloud SQL — not self-hosted in Kubernetes first

     Secrets                 Use External Secrets Operator or a secrets manager — never plain YAML in Git

     Static files            Serve via Nginx or CDN, never from Django runserver

     Media files             S3-compatible object storage — pods are ephemeral




     6.3 What NOT to Do First

        • Not Postgres in Kubernetes first — use a managed service; stateful K8s Postgres requires deep
        expertise
        • Not media storage inside pods — pods are ephemeral; files will be lost on restart or scale-down
        • Not secrets directly in YAML forever — use External Secrets or a secrets manager from day one
        • Not python manage.py runserver — this is the dev server; use Gunicorn in production
        • Not exposing Gunicorn directly — always put Nginx or an ingress in front
