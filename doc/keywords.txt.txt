📘 Kubernetes Keywords Guide (Industry-Focused)


---

🧩 1. apiVersion

✅ Common values:

v1 → Pod, Service, ConfigMap, Secret

apps/v1 → Deployment, StatefulSet, DaemonSet

batch/v1 → Job, CronJob

networking.k8s.io/v1 → Ingress, NetworkPolicy

autoscaling/v2 → HPA

rbac.authorization.k8s.io/v1 → Roles


🧠 Theory:

Defines which API group + version Kubernetes should use to understand your YAML.
Different resources belong to different API groups.

👉 Think: “Which part of Kubernetes handles this?”


---

🧱 2. kind

✅ Values:

Pod, Deployment, Service, Ingress

ConfigMap, Secret

Job, CronJob, StatefulSet


🧠 Theory:

Specifies what resource you are creating.

👉 Think: “What am I creating?”


---

🏷️ 3. metadata

✅ Fields:

name → unique name

namespace → default, dev, prod

labels → key-value pairs

annotations → extra info


metadata:
  name: my-app
  namespace: prod
  labels:
    app: my-app

🧠 Theory:

Holds identity and grouping information.

Labels are used for selection (VERY important)

Annotations are for extra data (not used for selection)


👉 Think: “Who is this resource?”


---

⚙️ 4. spec

🧠 Theory:

Defines the desired state of the resource.

Every resource has a different spec

Kubernetes tries to match reality to spec


👉 Think: “What do I want?”


---

🚀 5. replicas (Deployment)

✅ Values:

Any integer: 1, 2, 3, 10


🧠 Theory:

Number of Pod copies running simultaneously.

👉 Used for:

Scaling

High availability



---

🎯 6. selector

selector:
  matchLabels:
    app: my-app

🧠 Theory:

Connects Deployment/Service → Pods using labels.

👉 If labels don’t match → system breaks


---

🧬 7. template

template:
  metadata:
    labels:
      app: my-app
  spec:

🧠 Theory:

Blueprint used to create Pods.

👉 Deployment doesn’t create containers directly — it creates Pods via template


---

🐳 8. containers

containers:
  - name: app
    image: nginx:latest

🧠 Theory:

Defines actual application containers inside a Pod.

A Pod can have multiple containers (sidecars)



---

🌐 9. ports

ports:
  - containerPort: 80

🧠 Theory:

Tells Kubernetes: 👉 “This container listens on this port”


---

⚡ 10. env

env:
  - name: ENV
    value: prod

🧠 Theory:

Passes environment variables to containers.

👉 Used for configuration (like .env files)


---

🔐 11. envFrom

envFrom:
  - configMapRef:
      name: app-config
  - secretRef:
      name: app-secret

🧠 Theory:

Loads multiple environment variables from:

ConfigMap (non-sensitive)

Secret (sensitive)



---

📊 12. resources

resources:
  requests:
    cpu: "250m"
    memory: "128Mi"
  limits:
    cpu: "500m"
    memory: "256Mi"

🧠 Theory:

Controls CPU & memory usage.

requests → guaranteed minimum

limits → maximum allowed


👉 Prevents resource overuse & crashes


---

❤️ 13. livenessProbe

✅ Types:

httpGet

tcpSocket

exec


livenessProbe:
  httpGet:
    path: /health
    port: 80

🧠 Theory:

Checks if container is alive.

👉 If it fails → container is restarted


---

💚 14. readinessProbe

🧠 Theory:

Checks if container is ready to receive traffic.

👉 If it fails:

Pod is NOT restarted

Traffic is stopped



---

💾 15. volumes

✅ Types:

emptyDir

persistentVolumeClaim

configMap

secret


volumes:
  - name: data
    emptyDir: {}

🧠 Theory:

Defines storage available to Pod


---

📂 16. volumeMounts

volumeMounts:
  - name: data
    mountPath: /data

🧠 Theory:

Mounts volume inside container filesystem


---

🌐 17. Service.spec.type

✅ Values:

ClusterIP → internal (default)

NodePort → exposed via node

LoadBalancer → public IP (cloud)


🧠 Theory:

Defines how the app is exposed


---

🔌 18. Service.ports

ports:
  - port: 80
    targetPort: 8080
    nodePort: 30007

🧠 Theory:

port → service port

targetPort → container port

nodePort → external port (NodePort only)



---

🌍 19. Ingress

✅ apiVersion:

networking.k8s.io/v1


spec:
  ingressClassName: nginx
  rules:
    - host: myapp.com
      http:
        paths:
          - path: /
            pathType: Prefix

pathType values:

Prefix

Exact

ImplementationSpecific


🧠 Theory:

Routes domain → service

👉 Works like a reverse proxy


---

🧭 20. nodeSelector

nodeSelector:
  disktype: ssd

🧠 Theory:

Forces Pod to run on specific nodes


---

⚖️ 21. tolerations

tolerations:
  - key: "dedicated"
    operator: "Equal"
    value: "backend"
    effect: "NoSchedule"

🧠 Theory:

Allows Pods to run on tainted nodes


---

🔄 22. strategy

✅ Values:

RollingUpdate

Recreate


strategy:
  type: RollingUpdate

🧠 Theory:

Controls how updates happen


---

🔁 23. Rolling Update Fields

rollingUpdate:
  maxSurge: 1
  maxUnavailable: 0

🧠 Theory:

maxSurge → extra Pods allowed during update

maxUnavailable → Pods allowed to go down



---

🔐 24. securityContext

securityContext:
  runAsUser: 1000
  allowPrivilegeEscalation: false

🧠 Theory:

Controls container permissions & security


---

🔁 25. restartPolicy

✅ Values:

Always

OnFailure

Never


🧠 Theory:

Defines when container restarts

Can download URLs
